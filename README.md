# Задание 6

### List
Составить программу, которая в списке L заменяет первое вхождение списка L1 (если такое есть) на список L2.

### Алгоритм решения
Проверка входных данных: если какой-либо из списков (L, L1, L2) пустой или не существует, завершить работу с отрицательным результатом.
Поиск первого вхождения: проходим по основному списку L, начиная с первого элемента, и ищем подсписок, совпадающий с L1:
   1. Для каждой позиции i от 0 до (длина L минус длина L1) проверяем, совпадают ли элементы L[i], L[i+1], ... с элементами L1[0], L1[1], ...
   2. Если все элементы совпали - запоминаем позицию i и прекращаем поиск.
Замена - если вхождение найдено:
1. Удаляем из списка L элементы с позиции i в количестве, равном длине L1
2. Вставляем на это же место все элементы списка L2
3. Результат - возвращаем true, если замена выполнена, и false, если вхождение не найдено.

# Задание 7

### LinkedList
Отсортировать элементы списка по возрастанию.

### Алгоритм решения
1.Проверка списка - если список пустой или содержит 1 элемент, сортировка не требуется.

2. Повторяем следующие действия, пока происходит обмен элементов:
   - Устанавливаем флаг "обмен был" в false.
   
   - Проходим по списку от первого до предпоследнего элемента:
   
   - Сравниваем текущий элемент со следующим
   
Если текущий элемент больше следующего:
1. Меняем их значения местами
2. Устанавливаем флаг "обмен был" в true
Если за весь проход не было ни одного обмена, то сортировка завершена.

Результатом является список, отсортированный по возрастанию.

# Задание 8

### HashSet
Есть перечень компьютерных игр. Студенты группы играют в какие-либо из этих игр. Известно 
для каждого студента, в какие игры он играет. Определить: 
- в какие игры из перечня играют все студенты группы; 
- в какие игры из перечня играют некоторые студенты группы; 
- в какие игры из перечня не играет ни один из студентов группы?

### Алгоритм решения
1. Создание словаря студентов - для каждого студента хранится множество игр, в которые он играет.
2. Создание множества всех игр - полный перечень возможных игр.

Поиск игр, в которые играют ВСЕ студенты:
1. Берём множество игр первого студента как начальный результат
2. Для каждого следующего студента пересекаем текущий результат с его играми (оставляем только общие игры)
3. В итоге получаем игры, присутствующие у всех студентов

Поиск игр, в которые играют НЕКОТОРЫЕ студенты:
1. Создаём пустое множество
2. Для каждого студента добавляем все его игры в это множество
3. В итоге получаем все игры, в которые играет хотя бы один студент

Поиск игр, в которые НЕ играет НИ ОДИН студент:
1. Берём все игры из перечня
2. Удаляем из них те, в которые играют некоторые студенты
3. В итоге получаем игры, которые никто не выбрал

Вывод результатов - три списка игр с соответствующими пояснениями.

# Задание 9

### HashSet
Файл содержит текст на русском языке. Напечатать в алфавитном порядке все глухие согласные буквы, которые не входят хотя бы в одно слово.

### Алгоритм решения
1. Создание тестового файла: заполняем файл текстом на русском языке.
2. Чтение файла: считываем весь текст из файла.
3. Разбиение на слова: удаляем знаки препинания и разделяем текст на отдельные слова (пробелы, точки, запятые и т.д.).
4. Создание множества глухих согласных: определяем все глухие согласные буквы русского языка (п, ф, к, т, ш, с, х, ц, ч, щ).
5. Поиск встретившихся согласных:
   - Создаём пустое множество для найденных согласных
   - Для каждого слова переводим его в нижний регистр
   - Для каждой буквы в слове проверяем, является ли она глухой согласной
   - Если да - добавляем её в множество найденных согласных

Поиск отсутствующих согласных:
1. Берём множество всех глухих согласных
2. Удаляем из него те, которые были найдены в тексте
3. Сортировка и вывод:
   - Преобразуем результат в список и сортируем в алфавитном порядке
   - Выводим каждую отсутствующую согласную букву

# Задание 10

### Dictionary/SortedList
На городской олимпиаде по информатике участникам было предложено выполнить 3 задания, 
каждое из которых оценивалось по 25-балльной шкале. Известно, что общее количество 
участников первого тура олимпиады не превосходит 250 человек. На вход программы подаются 
сведения о результатах олимпиады.В первой строке вводится количество участников N. Далее 
следуют N строк, имеющих следующий формат:
<Фамилия><Имя><Баллы>
Здесь<Фамилия>– строка, состоящая не более чем из 20 символов;<Имя>– строка, состоящая не 
более чем из 15 символов;<Баллы>– строка, содержащая три целых числа, разделенных 
пробелом, соответствующих баллам, полученным участником за каждое задание первого тура. 
При этом <Фамилия> и <Имя>, <Имя> и <Баллы> разделены одним пробелом. Примеры входных 
строк: 
Петрова Ольга 25 18 16
Калиниченко Иван 14 19 15
Напишите программу, которая будет выводить на экран фамилию и имя участника, набравшего 
максимальное количество баллов. Если среди остальных участников есть ученики, набравшие 
такое же количество баллов, то их фамилии и имена также следует вывести. При этом имена и 
фамилии можно выводить в произвольном порядке.

### Алгоритм решения
1. Создаём текстовый файл с данными участников
Каждая строка содержит: фамилию, имя, три числа (баллы), разделённые пробелами

2. Открываем файл и читаем все строки. Каждую строку разбиваем на части по пробелам.

Из полученных частей извлекаем:
- Фамилию (часть 0)
- Имя (часть 1)
- Баллы (части 2, 3, 4), их преобразуем в числа

3. Для каждого участника складываем три балла, получаем общий результат участника

4. Поиск максимального балла
- Просматриваем всех участников
- Запоминаем наибольшую сумму баллов

5. Ещё раз просматриваем всех участников
Если сумма баллов участника равна максимальной - добавляем его в список победителей

Выводим максимальный балл

Выводим фамилии и имена всех победителей

Выводим количество победителей
